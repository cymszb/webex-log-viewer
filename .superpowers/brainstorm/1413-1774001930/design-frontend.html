Frontend Changes — index.html

Section 3 of 4 — What changes in the browser. Does this look right?

What's removed

  • Drag & drop — the drop overlay, FileReader handler, and client-side parseLine() function are all removed
  • "Load from Server" button — replaced by the upload panel below
  • All client-side log parsing logic — server now owns parsing entirely

What's added — Upload panel

Toolbar area (top of page)
No log loaded
✓ 42,310 lines loaded
  • File picker opens on "Upload Log…" click
  • Progress bar shown during upload (XHR with upload.onprogress)
  • On success: status shows line count; log table loads automatically via GET /api/logs
  • On error (413 too large, 400 bad file): clear inline error message
  • "Clear Log" calls DELETE /api/session/log and resets the table

What's unchanged

  • Log table, columns, pagination, level/PID/TID/source filters — identical
  • Full-text search and error navigation — identical
  • Topics sidebar (list view, edit view, groups, drag-and-drop) — identical
  • Events editor — identical
  • Topic Visualization timeline (canvas) — identical
  • All keyboard shortcuts — identical

Session awareness

  • On page load, GET /api/logs is called — if it returns {error:"no_log"}, the table shows an empty state with "Upload a log file to get started"
  • No login UI needed — the browser's session cookie handles identity transparently
  • If a user refreshes the page and their session is still valid, their log is still there
