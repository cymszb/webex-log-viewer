Configuration (env vars)

SECRET_KEY       # Flask session signing key (required in prod)
SESSION_DIR      # where session dirs are stored (default: ./sessions)
SESSION_TTL_H    # hours before idle session is cleaned up (default: 24)
MAX_UPLOAD_MB    # max log file size in MB (default: 200)

New / changed endpoints

Method Route Change
POST /api/upload NEW — accepts multipart file upload, saves to sessions/<id>/log.txt, returns {lines, size_mb}
GET /api/logs CHANGED — reads from sessions/<id>/log.txt instead of hardcoded current_log.txt. Returns 404 with {error:"no_log"} if no file uploaded yet.
DELETE /api/session/log NEW — lets user clear their uploaded log (frees disk space)
GET/POST/PUT/DELETE /api/topics
/api/groups		 UNCHANGED in structure — but all writes now go through filelock

Session management

  • On first request, Flask auto-creates a signed session cookie containing a session_id (UUID)
  • Each session gets a directory: SESSION_DIR/<session_id>/ with log.txt and meta.json (last-seen timestamp)
  • meta.json is touched on every /api/logs or /api/upload request
  • Background thread runs every hour, deletes session dirs where last_seen < now - TTL

File size limit

  • Flask's MAX_CONTENT_LENGTH set from MAX_UPLOAD_MB (default 200 MB)
  • Returns HTTP 413 if exceeded — frontend shows a clear error message
