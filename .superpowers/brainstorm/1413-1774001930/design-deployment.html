New project structure

File layout
WebexApp_Log_Viewer/
├── app.py                  # Flask backend (modified)
├── requirements.txt        # + filelock, gunicorn
├── .env.example            # documents env vars
├── sessions/               # created at runtime, gitignored
│   └── <session_id>/
│       ├── log.txt
│       └── meta.json
├── templates/
│   └── index.html          # frontend (modified)
├── topics.json             # shared config (unchanged format)
├── .gitignore              # add: sessions/, current_log*.txt
├── README.md               # updated
└── USER_GUIDE.md           # updated

Running locally (development)

pip install -r requirements.txt
python app.py                  # still works exactly as before

Running on a server (production)

# Set environment variables
export SECRET_KEY="change-me-to-a-random-string"
export SESSION_DIR="/var/lib/webex-log-viewer/sessions"
export SESSION_TTL_H=24
export MAX_UPLOAD_MB=200

# Start with gunicorn (4 workers)
gunicorn -w 4 -b 0.0.0.0:8098 app:app
  • Put nginx in front for SSL termination and static file caching
  • client_max_body_size in nginx must match MAX_UPLOAD_MB
  • SECRET_KEY must be set — app refuses to start without it in production mode
  • sessions/ directory must be writable by the gunicorn process

What does NOT change

  • topics.json format — fully backward compatible, existing config carries over
  • No database migration, no schema, no external services
  • Local single-user usage (python app.py) still works with zero config
