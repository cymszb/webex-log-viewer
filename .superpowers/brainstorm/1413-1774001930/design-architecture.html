Key design decisions

  • Session cookie вҖ” Flask's built-in signed cookie (SECRET_KEY). No DB needed. Session ID maps to a directory on disk.
  • Log isolation вҖ” each session gets its own sessions/<id>/log.txt. Users never see each other's data.
  • Shared topics вҖ” topics.json is still a flat file, protected by filelock for safe concurrent reads/writes.
  • TTL cleanup вҖ” a background thread checks session dirs every hour; deletes any idle longer than 24h (configurable).
  • Drag & drop removed вҖ” all log loading goes through POST /api/upload. Simplifies the frontend and eliminates the duplicate JS parser.
  • No auth вҖ” sessions are anonymous. Anyone with the URL can use the app.
  • Deployment вҖ” gunicorn + nginx. SECRET_KEY and session dir path set via environment variables.
